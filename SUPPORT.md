# Support

Thanks for using CloudREST. Here's how to get help, depending on what you need.

## I have a question about how to use CloudREST

Open a thread in [GitHub Discussions](https://github.com/ikeboy003/cloudrest/discussions). Questions like:

- "How do I express `<some query>` against this table?"
- "What's the right way to set up JWTs with my auth provider?"
- "Why is my RLS policy not matching?"

…belong in Discussions. The maintainers read them. Other users may too.

Please do **not** open an Issue for usage questions — Issues are reserved for bugs and feature requests.

## I found a bug

Open a [bug report](https://github.com/ikeboy003/cloudrest/issues/new?template=bug_report.md). Please include:

1. A minimal reproduction — the smallest request that shows the bug
2. What you expected to happen
3. What actually happened (full response body, not just the status code)
4. The CloudREST commit SHA, Postgres version, and Node version

Bugs with clean reproductions get fixed faster.

## I want a new feature

Open a [feature request](https://github.com/ikeboy003/cloudrest/issues/new?template=feature_request.md). Before you write code, it helps to discuss the design — feature requests that arrive as large PRs without prior discussion often need significant rework, which is frustrating for both sides.

## I think I found a security vulnerability

**Do not** open a public issue. See [SECURITY.md](SECURITY.md) for the private disclosure process.

## I want to contribute code

Read [ARCHITECTURE.md](ARCHITECTURE.md) first, then [CONTRIBUTING.md](CONTRIBUTING.md).
